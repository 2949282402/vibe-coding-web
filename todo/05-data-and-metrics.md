# 数据埋点与指标方案

## 一、目标

建立一套最小但有效的数据观测体系，用于判断：

- 用户是否真正使用知识问答
- 回答是否可信、是否有帮助
- Agent 产出是否被采纳
- 系统性能是否稳定

## 二、问答侧关键指标

### 使用指标

- 日问答次数
- 独立提问用户数
- 平均每会话提问轮数
- RAG / Ask 使用占比
- `LOCAL_ONLY` / `LOCAL_AND_WEB` / `WEB_ONLY` 使用占比

### 质量指标

- helpful rate
- 无命中率
- 低命中率
- 纯 Web Search 回答占比
- 含明确引用回答占比
- 会话 replay 使用率

### 性能指标

- 平均响应耗时
- P95 响应耗时
- SSE 打开成功率
- SSE 中断率
- 后端异常率

## 三、Agent 侧关键指标

### 产出指标

- Agent 任务数
- 草稿生成成功率
- 草稿转正式发布率
- 草稿被拒绝率
- 平均生成时长

### 质量指标

- 人工修改率
- 发布前退回率
- 发布后回滚率
- 审阅拦截率

## 四、后台看板建议

建议分三块卡片：

- 问答概览：问答量、帮助率、延迟
- 知识库质量：命中率、无命中问题、Web Search 依赖度
- Agent 生产质量：草稿数、审核通过率、发布时间

## 五、埋点落地建议

### 后端埋点优先

优先在后端落，因为后端更可靠，字段更完整。
建议埋点触发点：

- 提交问题时
- SSE 完成时
- 用户反馈 helpful / not helpful 时
- Agent 任务创建、完成、审核、发布时

### 前端埋点补充

前端补充埋点可记录：

- 模式切换行为
- 示例问题点击率
- 引用点击率
- 首页态模块点击率

## 六、验收标准

若完成本方案，至少应能回答以下问题：

- 用户主要是在问站内内容，还是在用泛问答？
- 哪些问题知识库答不好？
- Web Search 是否被过度依赖？
- Agent 生成的内容有多少真正被发布？
- 哪个环节最影响用户体验：命中率、可信度还是响应速度？
