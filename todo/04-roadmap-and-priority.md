# 迭代路线与优先级

## P0：必须先做

### 1. 拆分产品入口

目标：减少产品认知混乱。

任务：

- 从前台 `/knowledge` 中移除 `Agent` 一级入口
- 保留前台问答，后台独立承载 Agent
- 将 `Ask` 调整为高级模式，而不是主模式

产出：

- 新的页面信息架构
- 路由调整
- 模式切换逻辑调整

### 2. RAG 回答可解释性增强

目标：提升用户对答案的信任。

任务：

- 显示来源类型
- 显示命中来源数
- 显示可信度
- 显示耗时与主要依据内容
- 低置信度时增加明确提醒

### 3. Agent 改为草稿流

目标：降低自动发布风险。

任务：

- 取消默认直接发布
- 新增草稿状态
- 增加人工审核与发布动作

## P1：高价值增强

### 4. 知识库运营后台

目标：形成问答效果闭环。

任务：

- 展示问答总量、帮助率、平均耗时
- 展示无命中问题、低帮助率问题
- 展示 Web Search 使用占比
- 展示 SSE 失败与中断情况

### 5. Agent 内容生产工作台

目标：把现有零散后台能力整合为完整工作流。

任务：

- 当前任务列表
- 草稿审核队列
- 工具调用记录
- 已发布内容回溯

## P2：中期增强

### 6. 内容生产辅助能力

建议增加：

- 大纲生成
- 摘要生成
- SEO 描述生成
- 标题候选生成
- 对既有文章进行润色或重写

### 7. 问答首页推荐能力

建议增加：

- 热门问题
- 最近提问
- 与文章主题相关的引导问题
- 相关文章推荐

## P3：后续可选能力

### 8. 用户侧沉淀能力

可考虑：

- 收藏问答
- 保存对话为知识卡片
- 导出问答摘要

### 9. 内容分发能力

可考虑：

- 订阅更新
- 邮件摘要
- 面向不同渠道的内容改写

## 不建议当前阶段优先做的内容

- 不建议继续增加更多聊天模式
- 不建议优先投入复杂多 Agent 协同
- 不建议在没有运营数据前大改检索算法
- 不建议将网络搜索继续外扩为主能力

## 推荐排期

- 第 1 周：入口拆分、Ask 降级、Agent 前台移除
- 第 2 周：RAG 元信息与可信度展示
- 第 3 周：Agent 草稿流与审核动作
- 第 4 周：知识库运营后台 MVP
- 第 5 周及以后：内容生产增强和数据驱动优化
