# 第一阶段测试计划

## 一、目标

为第一阶段改造提供可执行的测试范围、测试路径和验收清单，确保产品结构调整、RAG 元信息增强、Agent 草稿流不会引入明显回归。

## 二、测试范围

第一阶段重点测试以下三类能力：

- `/knowledge` 页面产品结构调整
- RAG / Ask 响应元信息与 SSE 兼容
- Agent 草稿审核与发布流程

不在本阶段重点范围内：

- 检索算法效果深度优化
- 推荐、订阅等后续能力
- 大规模性能压测

## 三、前端测试项

## 1. `/knowledge` 页面

### 用例

- 首次进入页面时，显示首页空态、示例问题、能力说明。
- 普通用户看不到 `Agent` 主入口。
- Ask 不再作为首屏平铺主模式出现。
- 点击示例问题后可正常发起会话。
- 高级设置区可正常展开与收起。
- 搜索模式切换后，提问参数正确传递。

### 验收点

- 页面结构符合新的产品定位。
- 移动端和桌面端布局均可正常显示。

## 2. 问答结果展示

### 用例

- RAG 回答展示 `sourceType`、`confidenceLevel`、`latencyMs`、`retrievalHitCount`。
- Ask 回答展示 `sourceType=none` 或等价提示。
- `web` / `mixed` / `none` 三类来源文案正确。
- 低可信度结果显示警示提示。
- SSE 增量输出期间页面不抖动、不丢内容。

### 验收点

- 元信息展示不影响原有聊天体验。
- 元信息缺失时页面可容错。

## 3. 后台草稿审核页

### 用例

- 管理员可看到 Agent 草稿列表。
- 可进入草稿详情页或预览区。
- 可执行发布、驳回、返回编辑等动作。
- 状态变化后页面列表正确刷新。

### 验收点

- 后台可独立完成草稿审核闭环。

## 四、后端测试项

## 1. RAG / Ask 接口测试

### 普通接口

- `POST /api/public/rag/ask` 在 `RAG` 模式下返回新增字段。
- `POST /api/public/rag/ask` 在 `ASK` 模式下返回 `sourceType=none`。
- `LOCAL_ONLY`、`WEB_ONLY`、`LOCAL_AND_WEB` 三类模式字段正确。
- 弱召回或无召回时 `confidenceLevel` 合理。

### SSE 接口

- `POST /api/public/rag/ask/stream` 正常输出 `meta`、`delta`、`done`、`error`。
- `done` 事件返回完整 `AskResponse` 新结构。
- SSE 异常时能够输出 `error` 而非直接断流无响应。

## 2. Agent 审核流程测试

### 用例

- Agent 任务完成后进入 `DRAFT_READY`，而不是直接发布。
- 管理员调用审核通过接口后状态变为 `PUBLISHED`。
- 管理员调用驳回接口后状态变为 `REVIEW_REJECTED`。
- `FAILED`、`CANCELLED` 任务不能被发布。
- 非管理员访问审核接口返回权限错误。

### 验收点

- 状态机流转正确。
- 不存在绕过审核直接发布的路径。

## 3. 统计接口测试

### 用例

- 后台总览接口返回问答量、帮助率、平均耗时等字段。
- 无命中问题与低帮助率问题接口有稳定结构。
- 空数据场景不报错。

## 五、数据库测试项

- `rag_query_event` 可插入、可查询、可更新 helpful 状态。
- Agent 审核字段或审核表可正确保存审核人、时间、驳回原因。
- 初始化脚本可在新环境建表。
- 兼容升级逻辑可在旧环境补字段、补索引。

## 六、联调测试项

## 1. 前后端联调

- 新增 `AskResponse` 字段在前端可正确消费。
- 前端未升级完成前，后端新增字段不影响旧页面工作。
- 草稿审核接口与后台页面参数保持一致。

## 2. 部署联调

- nginx 下 SSE 依旧不缓冲。
- 前端改版后静态资源可正常构建与加载。
- 发布草稿后首页与文章列表缓存及时失效。

## 七、建议执行顺序

1. 先测后端接口新增字段是否稳定
2. 再测前端页面结构与元信息展示
3. 再测 Agent 草稿审核全链路
4. 最后做 docker / nginx / 缓存回归

## 八、最终验收清单

- 普通用户前台看不到 Agent。
- `/knowledge` 能清晰表达“基于本站知识回答”。
- RAG / Ask 回答均能提供正确的来源说明。
- Agent 生成内容默认进入草稿审核，不直接发布。
- 后台可完成草稿审核与发布。
- SSE、缓存、权限均未出现明显回归。
